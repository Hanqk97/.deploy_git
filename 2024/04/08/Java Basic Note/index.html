Data Type

  • Integer Types
    • byte: Occupies 1 byte of computer storage, representing a range from -128 to 127.
    • short: Takes up 2 bytes of storage, with a range from -32,768 to 32,767.
    • int: 4 bytes, with a range of approximately -2^31 to 2^31-1.
    • long: 8 bytes, with a range of approximately -2^63 to 2^63-1.
  • Floating Point Types
    • float: Single precision floating point, occupies 4 bytes, often used to store smaller floating point numbers.
    • double: Double precision floating point, takes up 8 bytes,typically used for storing larger or more precise floating point numbers.
  • Character Type
    • Used to store a single character (enclosed in single quotes), such as ‘A’ and ‘1’.
  • Boolean Type
    • Represents a boolean value, only two possible values: true and false.

Logical Operator

  • Logical AND operator &&: The entire result is true only if both conditions on the sides of the operator are true.
  • Logical OR operator ||: The result is true if at least one of the conditions on the sides of the operator is true.
  • Logical NOT operator !: This operator returns the opposite of the condition’s value, turning true to false and false to true.

If Condition

Array

  • Fixed Size: Once an array is declared, its size is usually fixed and cannot be dynamically changed during runtime.

  • Same Data Type: All elements in the array must be of the same data type.

  • Continuous Storage: Elements in an array are stored continuously in memory.

  • Index Access: Elements in an array are accessed through an index (or subscript).

  • Dynamic Initialization: After declaring an array, use the new keyword to allocate memory and initialize the elements.

  • Static Initialization: When declaring an array, memory is allocated, and initial values are specified simultaneously.

  • Length: All arrays possess a length attribute, which is used to obtain the length of the array, indicating the number of elements it contains.

printf()

Java offers a printf function for formatted output. It can output expressions args to the console in a specific format format. The syntax is as follows:

format is a string that specifies the output format.
args is a list of arguments that are inserted into the format string.

%d for integers
%f for floating-point numbers
%s for strings
%n for a newline character

Additionally, the format() method is equivalent to printf(). Both methods require a formatting string followed by arguments, where each argument corresponds to a format specifier.

When you need to output a floating-point number from integer division in Java, ensure that you cast one or both operands to double or float before performing the division.

Comparison of Character Sizes

  • Each character has a unique ASCII code value that represents its position in the ASCII character set. When comparing the size of characters, we are actually comparing their ASCII code values.
  • The difference between uppercase and lowercase letters is 32.
  • ASCII is an older character encoding standard that includes 128 characters such as basic Latin letters, numbers, and control characters, each associated with a unique integer. Unicode, on the other hand, is a comprehensive standard that covers nearly all characters and scripts globally with encodings like UTF-8, UTF-16, and UTF-32, which use different byte lengths. Essentially, ASCII is limited to common characters, while Unicode encompasses a vast range of characters from various languages.

String Operations

  • trim() Method: Removes whitespace from both ends of a string(front and back).
  • toCharArray() Method: Converts a string into a character array.
  • StringBuilder Class: Provides a mutable sequence of characters. It’s designed for use when you need to make a lot of modifications to strings, as it is more efficient than using immutable string objects.
    Key Methods:
    • append(): Adds characters or strings to the end of the StringBuilder instance, expanding the capacity if necessary.
    • toString(): Converts the StringBuilder back into a regular String object.
  • The reason to use StringBuilder over String for appending strings is due to the immutable nature of String objects. When you concatenate strings using String, each concatenation operation creates a new String object. This results in the creation of many temporary objects, which can negatively impact performance.

LinkedList I

LinkedList

  • Unlike arrays, the elements in a linked list in a computer’s memory can be either contiguous or non-contiguous. Each data element (referred to as a node) not only stores its own information (data) but also stores the address information (next pointer) of the next element.
  • The storage location of the first node in a linked list is called the head node. From the head node, the next node is found using the next pointer, and this continues until the last node is reached. The next pointer of the last node does not exist, meaning it is “null.” In Java, this is represented by null.

Defining Linked List and Linked List Nodes

  • In Java, one class can contain another (inner class). The inner class can access the private members of the outer class. Grouping related classes together can enhance the structure and readability of the code. Below, the Node class is placed inside the LinkedList class, indicating that Node is a part of LinkedList.

Doubly LinkedList

In Java, the LinkedList class implements a doubly linked list data structure. Each node in this structure contains data along with two references to the adjacent nodes: one pointing forwards and the other backwards.

add(Element): Adds an element to the end of the LinkedList.
add(index, Element): Inserts an element at a specified position, showcasing the high efficiency of LinkedList in insertion operations.
get(index): Retrieves the element at a specified index.
remove(index | Element): Removes an element at a specific position or a specific element.

Hash Tables

A hash table can be likened to a large dresser with many small drawers. Each drawer can be used to store various items.

  • Drawer Number: Each drawer has a number, serving as the key for storing and locating data.
  • Hash Function: This function converts a data’s key into a specific drawer number where the data should be placed.
  • Drawer Contents: Each drawer stores various items, representing the data.
  • Hash Collisions: If different keys result in the same drawer number, hash tables use certain methods to resolve these collisions.
  • Searching: To find data, hash tables quickly locate the appropriate drawer using the key and retrieve the contents.

Interfaces and Implementation Classes

  • Interface: An interface is an abstract data type that outlines a set of abstract methods. These methods typically lack an implementation; they’re only declared. The main role of an interface is to establish a standard for specific behaviors.

  • Implementation Class: This is a concrete class that implements the abstract methods defined in an interface. It provides actual implementations for these methods.

For example, in Java programming, Set is an interface that defines behaviors for collection classes, while HashSet is an implementation class that provides specific functionalities based on the Set interface.

Wash Dishes

Stack

A stack is a data structure that allows insertion and removal of elements only at one end, known as the top of the stack. This structure follows the Last In, First Out (LIFO) principle, meaning the most recently added element (the one closest to the top) is the first to be removed. The end where insertion and removal occur is called the top, and the opposite end is referred to as the bottom.

  • Import
  • Instance
  • Operations
    • isEmpty(): Determines if the stack is empty. Returns true if the stack is empty, otherwise returns false.
    • push(element): Adds an element to the stack. The new element becomes the top element of the stack.
    • pop(): Removes the top element from the stack and returns it.
    • peek(): Retrieves the top element without removing it from the stack.
    • size(): Returns the number of elements in the stack.

Queue for milk tea

Queue

A queue is similar to a queue or line of people waiting for a service. In a queue, new elements join at the end (also known as the rear or tail), and elements leave from the front (also known as the head or front). It performs insertions at the tail end (enqueue) and deletions at the head end (dequeue), adhering to the First-In-First-Out (FIFO) principle where the first element to enter the queue is the first to be removed.

  • Import And Instance
  • Operations
    • isEmpty(): Checks if the queue is empty. Returns true if it is, otherwise false.
    • add(): Enqueues an element by adding it to the tail of the queue.
    • poll(): Dequeues an element by retrieving and removing the head element of the queue.
    • peek(): Retrieves the head element without removing it from the queue.
    • size(): Returns the number of elements in the queue.
